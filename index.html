  • IoC / DI
  • @Conditional
  • @ConfigurationProperties
  • AutoConfiguration.imports

  • IService

  • Page 分页

学习路径

四次课的顺序不是简单并列，而是从框架机制、数据访问、安全控制到接口协作的递进。

1

理解框架为什么能少配置

从 Spring Boot 自动配置、条件装配和 Starter 入手，先弄清楚开箱即用背后的机制。

2

完成数据访问层闭环

把用户实体、查询参数、分页结果和 Controller 接口串起来，形成标准 CRUD 思路。

3

给接口加上身份边界

通过 Token、路由拦截、角色与权限注解，区分认证和鉴权两个不同问题。

4

让接口可以被协作和测试

使用 Swagger UI 展示、分组和调试接口，减少前后端沟通中的信息落差。

运行与接口测试

四个工程都以 Maven 启动为主，Day02 需要数据库，Day03 和 Day04 重点关注接口调用体验。

环境准备

  • JDK 17 或更高版本
  • Maven 3.6+
  • Day02 使用 MySQL 8.0，并创建 <代码>yyzx 数据库
  • Day04 默认 Swagger UI 地址：<代码 >http://localhost:8889/swagger-ui.html

常用接口

# Day02 分页查询
GET http://localhost:8888/yyzx/user/page?pageNum=1&pageSize=3

# Day03 管理员登录
POST http://localhost:8081/user/doLogin?username=admin&password=123456

# Day04 Swagger UI
http://localhost:8889/swagger-ui.html
